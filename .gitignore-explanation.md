# .gitignore Explanation

This document explains what files and directories are ignored by Git and why.

## Logs
```
logs/
*.log
npm-debug.log*
yarn-debug.log*
pnpm-debug.log*
```
**Why**: Log files contain runtime information that's environment-specific and can be regenerated. They shouldn't be committed.

## Dependencies
```
node_modules/
```
**Why**: Package dependencies are installed via `package.json` and `package-lock.json`. Committing `node_modules` (100k+ files) would bloat the repository.

## Build Outputs
```
dist/
dist-ssr/
*.local
```
**Why**: Build artifacts are generated from source code. They can be recreated with `npm run build` and differ per environment.

## Environment Variables
```
.env
.env.*
!.env.example
.dev.vars
```
**Why**:
- Environment files contain **sensitive credentials** (API keys, secrets)
- **SECURITY**: Never commit real credentials to Git
- `.env.example` is the ONLY env file tracked (template without real values)
- `.dev.vars` is Cloudflare's local dev environment file

## Cloudflare
```
.wrangler/
wrangler.toml.local
```
**Why**:
- `.wrangler/` contains local development state and cached D1 database
- `wrangler.toml.local` is for local overrides (may contain secrets)
- These are auto-generated during development

## Database (Local Development)
```
*.db
*.db-shm
*.db-wal
.turso
```
**Why**:
- Local SQLite database files from development
- Shared memory and write-ahead log files (SQLite internals)
- Production database is on Cloudflare D1 (cloud-hosted)

## Playwright (Testing)
```
playwright-report/
test-results/
playwright/.cache
```
**Why**:
- Test reports and results are generated per test run
- Can be regenerated with `npm run test`
- Cached browser binaries (~300MB) shouldn't be committed

## TanStack Router
```
.tanstack/
```
**Why**:
- Auto-generated route cache/metadata
- Regenerated during development and build

## Claude Code
```
.claude/
```
**Why**:
- Temporary files created by Claude Code CLI
- User-specific development artifacts

## Editor Directories
```
.vscode/*
!.vscode/extensions.json
.idea/
*.suo
*.njsproj
*.sln
```
**Why**:
- IDE-specific configuration files
- User preferences that differ per developer
- **Exception**: `.vscode/extensions.json` IS tracked (recommended extensions)

## TypeScript
```
*.tsbuildinfo
tsconfig.tsbuildinfo
```
**Why**:
- TypeScript incremental compilation cache
- Speeds up builds but environment-specific

## OS Files
```
.DS_Store
Thumbs.db
```
**Why**:
- macOS and Windows system files
- Created automatically by OS, not part of project

## Temporary Files
```
*.tmp
*.temp
.cache
```
**Why**: Generic temporary files that shouldn't persist

---

## What SHOULD Be Committed

✅ **Source Code**: `src/`, `db/schema.ts`
✅ **Configuration**: `package.json`, `tsconfig.json`, `vite.config.ts`, `wrangler.jsonc`
✅ **Database Schema**: `db/schema.ts`, `db/migrations/*.sql`
✅ **Documentation**: `README.md`, `db/README.md`
✅ **Templates**: `.env.example` (NO real credentials)
✅ **Tests**: `tests/`, `playwright.config.ts`

## What Should NEVER Be Committed

❌ **Secrets/Credentials**: API keys, passwords, tokens
❌ **Generated Files**: Build outputs, compiled code
❌ **Dependencies**: `node_modules/`
❌ **Local State**: `.wrangler/`, local databases
❌ **User-Specific**: IDE settings, OS files

## Quick Reference: Adding Secrets

**NEVER commit secrets!** Instead:

1. Add secret to `.env`:
   ```bash
   echo "SOLIDGATE_SECRET_KEY=abc123" >> .env
   ```

2. Verify it's ignored:
   ```bash
   git status  # .env should NOT appear
   ```

3. Document in `.env.example`:
   ```bash
   # Add placeholder to .env.example
   SOLIDGATE_SECRET_KEY=your_secret_key_here
   ```

4. For production (Cloudflare):
   ```bash
   wrangler secret put SOLIDGATE_SECRET_KEY
   ```

## Verify Ignored Files

Check if a file/directory is ignored:
```bash
git check-ignore -v .env .wrangler node_modules
```

See what's currently ignored:
```bash
git status --ignored
```
